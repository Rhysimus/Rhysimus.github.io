Bonk - an esoteric programming language

(1/12/2020)

What is this creation. I cannot believe this actually happened. What started as a gag turned into a genuine project - making an esoteric programming language.

It's essentially a programming language that is purposefully hard to follow and is not nice for us humans to work with. They go far away from traditional programming languages like C# and Java and instead enters the realm of madness (put lightly). Languages of these kind are Turing-complete, which means it can simulate a Turing machine (a system of rules, states and transitions) making the possibilities for such a language endlessly available to computational means.

I think it's crazy to think things like this can be done. The language that's been made here is a derivation of the esolang brainf*ck. What a name. It is possibly the most famous esolang, which is incredibly abstract. It was made by Urban Müller in 1993 to write the smallest compiler for Amiga OS. It works by operating on an array of memory cells and using a pointer it can do numerous functions. Low-level, sure, but that in itself can do a lot:

Command Description
> Move the pointer to the right
< Move the pointer to the left
+ Increment the memory cell at the pointer
- Decrement the memory cell at the pointer
. Output the character signified by the cell at the pointer
, Input a character and store it in the cell at the pointer
[ Jump past the matching ] if the cell at the pointer is 0
] Jump back to the matching [ if the cell at the pointer is nonzero

(table courtesy of Esolang)

Bonk essentially replaces brainf*ck's eight characters with eight strings, each with reference to the word "Bonk". It's strange what your brain comes up with at night, especially with other equally sleep-deprived individuals:

Command Description
BONK Move the pointer to the right
KNOB Move the pointer to the left
BONKBONK Increment the memory cell at the pointer
KNOBKNOB Decrement the memory cell at the pointer
BONKED Output the character signified by the cell at the pointer
KNOBED Input a character and store it in the cell at the pointer
BOINK Jump past the matching KNIOB if the cell at the pointer is 0
KNIOB Jump back to the matching BOINK if the cell at the pointer is nonzero

It plays with the word BONK, inverting it to KNOB for inverse operations. All commands are variations of this rule. KNOB isn't meant to have any negative connotations either, it just fit and made sense. To a sleep-deprived mind at least.

So you might say that this is just replacing commands and it's done. But it was actually quite a nightmare because of the different types of parsing. By no stretch of the imagination is Bonk meant to be small, compact and relatively efficient like brainf*ck but it is a sight to see when it works.

Anyway, back to the task at hand. The implementation that became the final version is a middleman-esque ploy on compiling. When you input Bonk code, it converts that into brainf*ck which it then runs as brainf*ck code. I'd have no problem wanting to make this so Bonk runs on its own compiler, but I feel like this way makes it so much more nicer. It also allows for equivalent conversions between the two languages (which is something I plan to extend to other languages because I think this is great). A start on this is up on the Bonk live demo which will be linked at the bottom of this post.

Right now this demo is able to convert brainf*ck into Bonk, and take Bonk code which runs within the browser. I've also made it so it can output things into a new window, which is nice for ASCII art or for forever calculating expressions. I've preloaded some files onto the website for people to load and play around with. These are mostly brainf*ck translations, but there is a few unique ones.

Surprisingly, I did actually learn a fair bit. The main thing is sleep is a virtue and I should get more of it, but the other thing is why esoteric languages of these kinds are actually really intuitive. The fact that it is Turing-complete is awesome - being able to play with the constraints of a low-level limited memory array while being able to input and output data is quite amazing. It made me do JavaScript too, and implement a fair few things here and there along with a comprehensible but simple UI on the demo to make this a genuine translator and compiler.

It also gave me an excuse to write a blog post. It's been a while, and I told myself I would do this more. ColourQR is getting there, I've been prototyping that a fair bit and there's a few things I've researched, realised and considered for making it great. More on that later. Thanks for reading.

[WARNING!] The demo works client-side and in pure JS, so as a result infinite loops do not return anything and that introduces the halting problem (without using Node.js or equivalent solutions to multithreading). So the worst this demo can do is crash your browser.
